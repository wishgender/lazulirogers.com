## Overview

I wanted to create a project for this portfolio, so I gave myself a two-week deadline and focused on two things I really enjoy: web apps and Pokémon.

## Key Features

- **Relational Database**: Allows easy fetching of events based on just month and day, but also streamlines the sorting process
- **Three-Day Display**: Displays events for the current day, as well as the previous and subsequent days
- **Theme toggle**: a dark mode/light mode toggle with color schemes by [Catppuccin](https://catppuccin.com).

## Technologies Used

- **libSQL3**: Database is hosted by Turso, who maintain libSQL, an open-source, open-contribution alternative to SQLite
- **React**: The front-end components were built using React.js
- **Styled-Components**: Much of the styling for this project was done via Styled-Components, to ensure each component looks exactly as I want it to

## Challenges & Lessons Learned

An issue I often find myself facing is that I don't entirely plan out the project beyond the baseline ideas, so I tend to end up making many changes throughout the design process, resulting in code that can be confusing at best, and spaghetti-code at worst. With this project, I determined the minimum functionality I wanted to see in a finished product and focused on just implementing that. However, I didn't really apply that philosophy to the front-end design and ended up having some difficulty with that part of the project. In future projects, I now plan to have a baseline design for both the front-end *and* the back-end before going into it.

## Outcome

I'm quite happy with this project. I'm considering revamping it with some images and perhaps the ability to browse by date in the future, but for now, it's a satisfying result.
